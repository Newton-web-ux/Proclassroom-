Before you start a class, a few notes

  • A class join code lets anyone who has it join as a student. Treat it like a door key — share it with your class, not in public.
  • Pro Classroom needs an internet connection to sync; drawings and posts made offline won't save until you're back online.
  • Deleting a class removes its assignments, grades, stream posts, and submissions permanently. There's no recovery after deletion.
  • Pro Classroom is an independent tool and isn't affiliated with, or endorsed by, Google. Sign-in and file storage run on your own Google account.
  • This is built by a solo developer, not a school district vendor — see in the user guide before relying on it for graded coursework.

Privacy policy

Last updated: 17 July 2026 · Applies to the Pro Classroom web application

What we collect

Pro Classroom collects only what it needs to run a class:

  • Account info from Google Sign-In: your name, email address, and profile picture.
  • Class content you create: class names, announcements, assignments, grades, submissions, and whiteboard drawings.
  • Files you attach: images and documents uploaded to assignments, posts, or submissions.
  • Basic usage data, such as when you last opened the app, used only to keep sync reliable.

We do not collect payment card details directly — Pro plan payments are handled through a UPI transfer you initiate, verified manually.

How it's used

Your data is used strictly to operate the class you're part of: displaying the stream, storing grades, syncing the whiteboard in real time, and letting a teacher's Pro plan cover the classes they've created. We do not use your class content to train any model, and we do not sell data to third parties.

Where it's stored

Class data (posts, assignments, grades, whiteboard strokes) is stored in Google Firestore under Pro Classroom's Google Cloud project. File attachments are stored in Google Drive, organized into a folder per class. Both are Google infrastructure; Pro Classroom does not run its own separate servers for this data.

Sharing

Within a class, teachers can see everything posted in that class, including all students' submissions and grades. Students can see the stream and classwork, and only their own grades and submissions — never another student's. We don't share your information with advertisers, and Pro Classroom does not display third-party ads.

Your controls

  • You can delete any attachment, post, or submission you created at any time.
  • A teacher can permanently delete a class, which removes its assignments, grades, stream, and submissions.
  • You can request full deletion of your account data by contacting us — see below.

Students & minors

Pro Classroom is intended to be set up by a teacher or another adult, who is responsible for confirming it's appropriate for their students' ages and for their school's policies before use. If you're a parent or guardian with questions about a student's data, contact us using the details below.

User guide

Everything you need to run your first class.

Getting started

Pro Classroom opens straight to a Google sign-in screen. Use the same account you'd normally use for school — there's no separate Pro Classroom password to create or lose.

1

Sign in with Google

Grant sign-in permission when prompted. Your name and photo appear in the top bar once you're in.

2

Choose your path

Teaching? Create a class. Joining someone else's? Use "Join with code" from the sidebar instead.

3

You're in the dashboard

From here every class you teach or belong to appears in the left sidebar.

For teachers

Creating a class: click "+ Create class," give it a name, section, room, and color. A six-character join code is generated automatically — find it any time at the top of the class page.

Posting to the stream: open a class → Stream tab, write your update, optionally attach a file or link, and post. It reaches every student instantly.

Creating classwork: Classwork tab → "+ Create assignment." Set a due date and total marks; students see a countdown and submit their work back to you.

Grading: the Grades tab shows a spreadsheet — click any cell to enter a score. Averages calculate automatically.

Managing your roster: the People tab lists everyone who's joined with your code. You can also add roster-only entries manually if a student hasn't joined yet.

For students

Joining a class: from the dashboard, click "Join with code" and enter the code your teacher shared. You'll land on the class stream immediately.

Submitting classwork: Classwork tab → open an assignment → "Submit work." Attach a file, an image, or a link, add an optional note, and submit. You can resubmit any time before — or after — the due date.

Checking your grades: your Grades tab only ever shows your own scores, never the rest of the class.

Using the whiteboard

Each class can have multiple whiteboards, and each whiteboard can have multiple pages. Only the teacher draws — students watching automatically follow whichever board and page the teacher currently has open.

Teachers: open the Whiteboard tab → "+ New board" to start one. Use the toolbar to change pen color, thickness, or switch to the eraser. Undo/redo work like you'd expect, and "Clear all" wipes the current page for everyone.

Students: if you navigate away from the live page to look at an earlier one, use "Jump to live" to snap back to whatever the teacher is currently showing.

Cautions

A few things worth knowing before you rely on Pro Classroom day-to-day:

  • Join codes aren't password-protected beyond the code itself — anyone with it can join as a student. Don't post it somewhere public.
  • Deleting a class or an assignment is immediate and permanent. There's no undo or recycle bin.
  • The app needs a live connection to sync; don't count on offline edits saving automatically.
  • Large image attachments are automatically compressed — if you need the original, full-resolution file, keep a separate copy yourself.

Support & limits

Pro Classroom is built and maintained by a single independent developer rather than a support team, so response times may vary — but issues are read and taken seriously.

Because storage and sync run on ordinary Google Cloud infrastructure rather than a dedicated enterprise backend, Pro Classroom is best suited to individual classes and small schools rather than large district-wide rollouts. For anything storing graded coursework that matters, we'd recommend keeping your own backup copy of key files.
